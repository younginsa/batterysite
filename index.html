How it works

From input to insight in three steps

The interface stays simple while the solver does the heavy lifting on the server. You never touch the compute layer.

01

Pick & set conditions

Choose a cell from the library, then set C-rate, ambient temperature, and capacity through the web controls.

02

Run on the server

A containerized solver — your C, MATLAB, or Python code — receives the parameters, executes, and writes structured results back.

03

Read the results

Voltage curves and thermal contour plots render in the browser with key metrics. Export the raw data anytime.

Service structure

From physical cell to digital twin

Three stages turn a battery on the bench into a model you can run in the browser. Click each stage to see what happens inside.

STAGE 01

Testing

Charge–discharge cycling and characterization of the received cell.

STAGE 02

Teardown analysis

Disassemble, measure every model input, digitize the cell.

STAGE 03

Model

Drive the physics model with measured data — the digital twin.

Deliverables

The first model package

◷ First version targeted — end of August

Every model ships as a complete, runnable set. Open each item to see what's included.

01Model execution code / executable▾
Solver source or a packaged executable — the same physics code we run server-side, containerized so it behaves identically on any machine.
02Input TXT + parameter guide▾
A plain-text input deck listing every parameter with its meaning, unit, and valid range — edit values, rerun, and compare.
03Sample output set▾
Reference results for a known case, so you can confirm your run reproduces ours before trusting new cases.
04Run & verification procedure▾
A step-by-step procedure: execute the model, read the outputs, and validate them against the sample set.
